1. Welcome

Cheradip is an AI coding assistant that sits in your editor activity bar. It can:

  • 💬 Chat about your code with streaming replies
  • 🤖 Agent mode — read/write files, run terminal commands, create projects
  • 🧩 Composer — multi-step edits across the workspace
  • 📎 @ context — attach files and search the codebase
  • ✏️ Inline edit — change a selection in place (Ctrl+K style)
  • 🏠 Use Cheradip AI by default (no API key), or your own cloud / Ollama keys

Requires VS Code 1.85+ (or another VS Code–compatible editor).

2. Install

From the Marketplace (recommended)

  1. Open VS Code → Extensions (Ctrl+Shift+X)
  2. Search for Cheradip (publisher cheradip)
  3. Click Install, then reload if prompted
  4. Look for the Cheradip icon in the left activity bar

From a VSIX file

  1. Get a .vsix package from your team or build it yourself
  2. Extensions → ⋯ → Install from VSIX…
  3. Or CLI: code --install-extension path\to\cheradip-x.y.z.vsix

Build from source (developers)

cd path\to\cheradip
npm install
npm run build
npx @vscode/vsce package --no-dependencies
code --install-extension cheradip-*.vsix

After install: Developer: Reload Window.

3. First run

  1. Click the Cheradip activity-bar icon to open the sidebar chat.
  2. Optionally sign in: Command Palette → Cheradip: Sign In (needed for paid plans and synced billing).
  3. Open settings or the config file if you want a cloud API key or a custom Cheradip AI URL.
  4. Confirm the status line shows something like Connected · ai.cheradip.com (or your chosen provider).
  5. Type a question and send with Enter (Ctrl+Enter or Shift+Enter inserts a new line).

Tip: Open a folder as a workspace before using agent file tools — paths are relative to the workspace root.

4. Chat & sessions

  • Session tabs — each chat is a tab; use + for a new session
  • Streaming — replies arrive progressively; use ■ Stop or Cheradip: Stop Agent to cancel
  • Copy — copy buttons on messages show a brief “Copied” snackbar
  • Clarifying questions — when the agent is unsure, clickable options may appear; pick one to continue
  • History — conversation memory stays in your local profile/session (not synced across machines by default)

Commands: Cheradip: Open Chat, Cheradip: New Chat Session, Cheradip: Focus Chat Input (Ctrl+L).

6. Agent mode

With cheradip.agentMode enabled (default), Cheradip runs an autonomous agent loop through tools until the task is done.

What the agent can do

  • List the project, search text, read and write files
  • Create / delete files and folders
  • Run shell commands (runCommand captures output; runTerminal opens a visible terminal)
  • Create a starter project layout

Permissions

Each tool action asks for approval:

  • Allow — run this action once
  • Deny — skip

There is no session-wide “allow all”. Use Cheradip › Agent: YOLO mode to auto-approve. Reset legacy cache: Cheradip: Reset Agent Permissions.

Prompt preparation

Like Cursor, Cheradip can analyze your request with a fast model before the main agent runs. Planning streams in a temporary panel (not saved); the Execution plan appears in the reply and guides the agent. Toggle: cheradip.promptReadyEnabled · prep model: cheradip.promptReadyModel.

Safeguards

  • cheradip.agentMaxIterations — max tool loop steps (default 12)
  • cheradip.agentYoloMode — skip prompts (use carefully)
  • cheradip.agentDiffReview — review diffs before writes

7. Composer

Composer is the focused “get this change done” flow:

  • Cheradip: Open Composer (Agent) — agent-style multi-step work
  • Cheradip: Open Composer Panel — editor-panel variant
  • Cheradip: Run Background Agent — longer jobs with a separate iteration limit

Related settings: cheradip.composerMaxIterations, cheradip.backgroundAgentMaxIterations, cheradip.chatMode.

8. @ context & search

  • Cheradip: Insert @ Context — mention files / symbols for the next message
  • Cheradip: Add File to Context — from the editor context menu
  • 📎 attach button in the composer
  • Codebase search — lexical or embedding modes (cheradip.codebaseSearchMode)
  • Cheradip: Build Semantic Index / Clear Semantic Index / Refresh Codebase Index

Embedding backend is configurable via cheradip.embeddingBackend.

9. Inline edit & apply

  • Inline Edit (Ctrl+K) — select code → instruct Cheradip to rewrite the selection
  • Ghost completions — optional inline suggestions (cheradip.inlineCompletion)
  • Apply on fenced code in chat — write the snippet into the workspace (Cheradip: Apply Code)

Editor shortcuts: Ctrl+Shift+K (inline edit), right-click → Inline Edit / Add File to Context.

10. Checkpoints & diffs

  • With cheradip.agentCheckpoints, Cheradip can snapshot files before agent writes
  • Cheradip: Restore Checkpoint (pick) — roll back a snapshot
  • With cheradip.agentDiffReview, review side-by-side diffs and Accept / Reject before applying

11. Sign-in & account

Extension accounts are separate from the Cheradip MCQ / Android app phone login.

  • Cheradip: Sign In — username/email + password for the extension App API
  • Cheradip: Sign Out / Sign Out All Devices
  • Cheradip: Enter License Key — if you use offline/license activation

API base (default): https://cheradip.com/ailt/api/ (cheradip.appApiUrl).

15. MCP & rules

User rules (cheradip.userRules) are injected into every agent turn — great for house style, “always use TypeScript”, security constraints, and so on.

MCP servers (cheradip.mcpServers) let you attach Model Context Protocol tools, for example a filesystem MCP:

{
  "filesystem": {
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path"]
  }
}

Native tool-calling preference: cheradip.nativeToolCalls.
