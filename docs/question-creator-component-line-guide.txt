================================================================================
  PART A — THE STRUCTURED HEADER TEXTAREA (questionHeader)
================================================================================

  Storage
  -------
  • Raw string: property questionHeader (line ~131). Newlines separate rows.
  • Parsed rows for preview logic: getter headerPreviewLines (lines ~1675–1677).
    Splits on \n, trims ends only; includes blank lines if present.

  Legacy vs structured
  --------------------
  • headerUseLegacyQuestionHeader (line ~138): true = free-form textarea only;
    false = EIIN + exam type drive first two lines via rebuildQuestionHeader.
  • paperSubjectMetaLinesEligible (lines ~1682–1687): structured “subject meta”
    features (MCQ title rows, unified mixed layout, etc.) need subject + questions.

  Default HR token
  ----------------
  • DEFAULT_STRUCTURED_HEADER_HR = '<hr>' (lines ~208–212). Inserted in several
    sync paths for MCQ-only / creative-only layouts (see comments there).


--------------------------------------------------------------------------------
  A.1  Mixed CQ + MCQ (two sheet kinds, separate preview pages)
--------------------------------------------------------------------------------
  Flag: mixedHeaderUsesExpandedEditorLines (lines ~1720–1726).

  Constants (lines ~375–388):
  • MIXED_HEADER_UNIFIED_MIN_LINES = 7  — minimum unified textarea rows.
  • MIXED_HEADER_UNIFIED_SQ_NOTICE_LINES = 9 — when subject sq is 25 or 30.

  Intended mapping (0-based indices in questionHeader / headerPreviewLines):
  • [0] Institute line (name, district).
  • [1] Exam line (label − year/serial); subject code often in brackets here.
  • [2] Plain subject name (mixed: no “(সৃজনশীল)” suffix in textarea; preview adds).
  • [3] MCQ combined সময় + পূর্ণমান (one textarea row; preview may split visually).
  • [4] সৃজনশীল combined সময় + পূর্ণমান.
  • [5] Horizontal rule row (<hr>) in storage (may be filtered from sidebar list).
  • [6] Plain “বিষয় কোড” line for both grids — mixedUnifiedCodeGridPlainLine (1735).
  • [7] CQ দ্রষ্টব্য (sq 25/30 only, when nine lines).
  • [8] MCQ দ্রষ্টব্য (sq 25/30 only).

  Canonical notice strings: MIXED_SQ_NOTICE_* (lines ~390–397).

  Key functions:
  • ensureMixedExpandedHeaderEditorLines — lines ~1983–2064 (pad, migrate legacy).
  • normalizeMixedExpandedSevenLineHeader — lines ~1924–1936.
  • migrateMixedHeaderNineLinesToSix / UnifiedSixToSeven — ~1821–1907.
  • syncMcqSqMetaMixedTextarea — lines ~2152–2185 (seed MCQ/CQ meta lines).
  • syncMixedSq25Sq30NoticeLines — lines ~2377+ (sq notices).
  • mixedHeaderShiftCodeBeforeSqNoticesIfNeeded — lines ~2350+ (repair code index).

================================================================================
  PART B — SIDEBAR HEADER EDITOR vs questionHeader LINE INDICES
================================================================================

  getHeaderEditorLinesRaw (lines ~3764–3771)
  • Drops lines that are only <hr> (case-insensitive substring) OR blank.
  • So physical line index in questionHeader ≠ “model index” in the sidebar.

  headerEditorModelIndexToQuestionHeaderLineIndex (~3778–3788)
  • Maps visible sidebar row order back to the real split index.

  headerEditorLineHiddenFromSidebar / getHeaderEditorSidebarRows (~3795–3808)
  • Mixed + sq: duplicate CQ notice row can be hidden from sidebar while kept
    in questionHeader for PDF/sync.

  Per-line fonts: headerLineFontSizes (line ~152); length tracks questionHeader
  lines after syncHeaderFontSizesToLineCount (~3709+).

  User edits: onHeaderEditorLineChange, onHeaderEditorLineKeydown (~3835–3873).
  Font ±: onHeaderLineFontAdjustInEditor, dec/incHeaderLineFontAt (~3884–3922).
